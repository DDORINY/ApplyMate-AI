# 권한 정책

## 기본 원칙

ApplyMate AI의 모든 사용자 데이터는 기본적으로 사용자 본인만 접근할 수 있다. 관리자 기능과 다중 사용자 협업은 현재 범위가 아니므로 사용자 간 데이터 공유를 제공하지 않는다.

## 인증 상태

| 상태 | 설명 | 허용 기능 |
| ---- | ---- | --------- |
| 미인증 | 로그인하지 않은 사용자 | 회원가입, 로그인, Health API |
| 인증 | 유효한 Access Token을 가진 사용자 | 본인 정보 조회, 향후 본인 데이터 관리 |
| 비활성 | `INACTIVE` 상태 사용자 | 로그인 및 보호 API 접근 불가 |
| 탈퇴 | `WITHDRAWN` 상태 사용자 | 로그인 및 보호 API 접근 불가 |

## v0.1.1 보호 API

| API | 인증 필요 | 권한 기준 |
| --- | --------- | --------- |
| `GET /api/v1/health` | 아니오 | 공개 상태 확인 |
| `POST /api/v1/auth/signup` | 아니오 | 신규 계정 생성 |
| `POST /api/v1/auth/login` | 아니오 | 이메일과 비밀번호 검증 |
| `POST /api/v1/auth/refresh` | 예 | 유효한 Refresh Token Cookie |
| `POST /api/v1/auth/logout` | 조건부 | Refresh Token Cookie가 있으면 폐기 |
| `GET /api/v1/auth/me` | 예 | 유효한 Access Token |

## 사용자 소유권

향후 프로필, 공고, 지원 문서, 일정 API는 인증된 사용자의 `user_id`를 기준으로 데이터 접근을 제한한다.

원칙:

* 사용자는 본인 데이터만 조회, 생성, 수정, 삭제할 수 있다.
* URL path의 id만 신뢰하지 않고 DB 조회 시 `user_id` 조건을 함께 적용한다.
* 존재하지 않는 데이터와 접근 권한이 없는 데이터의 오류 메시지는 과도한 정보를 노출하지 않는다.

## 외부 서비스 권한

Google Calendar, Gmail 등 외부 서비스 연동은 사용자의 명시적 OAuth 승인 후에만 수행한다. 연동 해제 시 이후 동기화는 중단하며, 기존 내부 데이터 삭제 여부는 별도 정책으로 정한다.

# 외부 API 연동

## 개요

외부 API 연동은 MVP 후반부에서 도입한다. 현재 v0.1.1에서는 실제 외부 API 호출을 구현하지 않는다. 필요한 환경변수 이름만 `.env.example`에 정의해 이후 확장을 준비한다.

## 연동 후보

| 서비스 | 목적 | 예정 버전 |
| ------ | ---- | --------- |
| Google Calendar API | 지원 일정 외부 캘린더 등록 및 동기화 | v0.5.0 |
| Gmail API | 채용 관련 이메일 분석 및 일정 후보 추출 | v0.5.1 |
| OpenAI API | 채용공고 분석, 자기소개서 생성, 면접 질문 생성 | v0.2.x 이후 |
| GitHub API | 사용자 프로젝트 활동 참고 자료 수집 | v1.0 이후 검토 |
| 채용공고 공개 API | 공고 수집 | v0.6.x |

## 공통 원칙

* 사용자의 명시적 승인 없이 외부 계정을 연결하지 않는다.
* OAuth Token과 API Key는 저장소에 커밋하지 않는다.
* 외부 API 장애가 내부 데이터 저장 실패로 번지지 않도록 처리한다.
* 외부 API 응답은 필요한 필드만 저장한다.
* 외부 서비스 이용약관과 rate limit을 준수한다.

## Google Calendar 연동 정책

* 일정 등록 전 사용자의 OAuth 연결 상태를 확인한다.
* 내부 일정 저장이 우선이며 외부 동기화는 부가 작업으로 처리한다.
* 동기화 실패 시 사용자에게 재시도 가능한 상태를 제공한다.
* 확정 일정과 예상 일정을 구분해 캘린더에 표시한다.

## Gmail 연동 정책

* 사용자의 승인 범위 내에서 채용 관련 메일만 분석한다.
* 메일 원문 전체 저장을 기본으로 하지 않는다.
* 일정 후보는 자동 확정하지 않고 사용자 확인 후 저장한다.

## OpenAI API 연동 정책

* 사용자의 실제 경험을 근거로 사용한다.
* 근거가 부족한 내용을 사실처럼 생성하지 않는다.
* 프롬프트와 응답 스키마는 기능별로 관리한다.
* API Key는 환경변수로만 주입한다.

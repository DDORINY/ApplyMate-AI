# 내비게이션 구조

## 현재 구현 화면

v0.1.2 기준 실제 구현된 화면은 다음과 같습니다.

| 경로 | 설명 | 상태 |
| --- | --- | --- |
| `/` | 서비스 상태와 주요 화면 진입점 | 완료 |
| `/signup` | 회원가입 | 완료 |
| `/login` | 로그인 | 완료 |
| `/me` | 현재 로그인 사용자 정보 | 완료 |
| `/profile` | 커리어 프로필 관리 | 완료 |

## 홈 화면 진입점

홈 화면은 다음 버튼을 제공합니다.

- 회원가입
- 로그인
- 내 계정
- 프로필

## `/profile` 화면 구성

`/profile`은 v0.1.2 범위를 한 화면 안에서 섹션 형태로 관리합니다.

섹션:

- 기본 프로필
- 기술 스택
- 경력
- 프로젝트
- 희망 조건
- 지원 제외 조건
- 포트폴리오 링크

## 인증 흐름

- `/signup`, `/login`은 비인증 사용자도 접근할 수 있다.
- `/me`, `/profile`은 인증이 필요하다.
- 인증되지 않은 사용자가 보호 화면에 접근하면 로그인 화면으로 이동한다.
- API 클라이언트는 Access Token 만료 시 Refresh Token 재발급 흐름을 재사용한다.

## 모바일 기준

- 한 화면을 세로 스크롤 중심으로 구성한다.
- 입력 섹션은 카드형 패널로 분리한다.
- 삭제 액션은 확인 후 실행한다.
- v0.1.2에서는 복잡한 탭/단계형 UI보다 빠른 입력과 검증을 우선한다.

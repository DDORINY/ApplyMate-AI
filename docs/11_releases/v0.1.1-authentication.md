# ApplyMate AI v0.1.1

## 1. 버전 개요

v0.1.1은 이메일 기반 회원가입/로그인과 JWT 인증을 구현한 버전이다. 사용자 계정과 refresh token 저장 구조가 추가되어 이후 모든 사용자 소유 데이터 기능의 기반이 되었다.

## 2. 개발 배경

ApplyMate AI의 모든 핵심 데이터는 사용자별로 분리되어야 한다. 따라서 커리어 프로필과 채용공고 기능 전에 인증과 사용자 식별 체계를 먼저 구현했다.

## 3. 해결하려는 문제

- 사용자 계정 부재
- 보호 API 호출 방식 부재
- Refresh Token 재사용/폐기 정책 부재
- Frontend 로그인 상태 관리 부재

## 4. 사용자 가치

사용자는 이메일로 가입하고 로그인한 뒤 자신의 계정으로 서비스 기능을 사용할 수 있다.

## 5. 구현 범위

- 이메일 회원가입
- 이메일 로그인
- JWT Access Token 발급
- Refresh Token HttpOnly Cookie 발급
- Refresh Token DB hash 저장
- Refresh Token rotation
- 로그아웃
- 현재 사용자 조회
- 회원가입/로그인 화면

## 6. 제외 범위

- 이메일 인증
- 비밀번호 찾기/재설정
- 소셜 로그인
- 세션 목록 관리
- 계정 삭제

## 7. 시스템 변경사항

- 인증 라우터 추가
- User/RefreshToken 모델 추가
- 인증 dependency 추가
- Frontend token 저장 및 보호 화면 연결

## 8. Backend 구현

- `auth.py` endpoint
- 사용자 생성/로그인 service
- 비밀번호 hash
- JWT 생성/검증
- Refresh Token hash 저장과 rotation
- `get_current_user` dependency

## 9. API 변경

| Method | Path | 인증 | 설명 |
| --- | --- | --- | --- |
| POST | `/api/v1/auth/signup` | 불필요 | 이메일 회원가입 |
| POST | `/api/v1/auth/login` | 불필요 | 이메일 로그인 |
| POST | `/api/v1/auth/refresh` | Refresh Cookie | Access Token 재발급 |
| POST | `/api/v1/auth/logout` | Refresh Cookie | Refresh Token 폐기 |
| GET | `/api/v1/auth/me` | Access Token | 현재 사용자 조회 |

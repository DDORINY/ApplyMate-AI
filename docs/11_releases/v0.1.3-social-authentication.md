# ApplyMate AI v0.1.3

## 1. 버전 개요

v0.1.3은 Google/GitHub OAuth 로그인과 소셜 계정 연결 기능을 구현한 버전이다.

## 2. 개발 배경

사용자가 이메일/비밀번호 외에도 소셜 계정으로 로그인할 수 있어야 서비스 진입 장벽이 낮아진다.

## 3. 해결하려는 문제

- OAuth authorize/callback 흐름 부재
- OAuth CSRF 방어 state 부재
- provider callback 후 frontend token 교환 흐름 부재
- 소셜 계정 연결/해제 정책 부재

## 4. 사용자 가치

사용자는 Google 또는 GitHub 계정으로 로그인하거나 기존 계정에 소셜 계정을 연결할 수 있다.

## 5. 구현 범위

- Google OAuth
- GitHub OAuth
- OAuth provider 상태 API
- authorize URL 생성
- callback 처리
- one-time login ticket exchange
- 소셜 로그인 사용자 생성
- 기존 이메일 계정 자동 병합 방지
- 명시적 계정 연결
- 계정 연결 해제
- 마지막 로그인 수단 보호

## 6. 제외 범위

- Kakao/Naver/Apple 로그인
- provider access token 저장
- Gmail/GitHub repository 데이터 수집
- Google Calendar 연동

## 7. 시스템 변경사항

- `auth/oauth` 라우터 추가
- OAuth state/ticket 기반 보안 흐름 추가
- Frontend OAuth callback 화면 추가

## 8. Backend 구현

- `OAuthAccount`
- `OAuthState`
- `OAuthLoginTicket`
- OAuth provider enum
- state hash 저장
- ticket hash 저장
- provider token 미저장 정책

## 9. API 변경

| Method | Path | 설명 |
| --- | --- | --- |
| GET | `/api/v1/auth/oauth/providers` | provider 활성 상태 |
| GET | `/api/v1/auth/oauth/{provider}/authorize` | 로그인 authorize URL |
| GET | `/api/v1/auth/oauth/{provider}/link/authorize` | 계정 연결 authorize URL |
| GET | `/api/v1/auth/oauth/{provider}/callback` | provider callback |
| POST | `/api/v1/auth/oauth/exchange` | one-time ticket 교환 |
| GET | `/api/v1/auth/oauth/accounts` | 연결 계정 목록 |
| DELETE | `/api/v1/auth/oauth/accounts/{provider}` | 연결 해제 |

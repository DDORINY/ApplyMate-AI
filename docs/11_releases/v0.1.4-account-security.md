# ApplyMate AI v0.1.4

## 1. 버전 개요

v0.1.4는 계정 보안·복구 기능을 구현한 버전이다. 이메일 인증, 비밀번호 복구, 세션 관리, 보안 이벤트 기록이 추가되었다.

## 2. 개발 배경

기본 인증 이후 운영 가능한 계정 시스템이 되려면 이메일 검증, 비밀번호 복구, 세션 추적, 로그인 실패 제한이 필요하다.

## 3. 해결하려는 문제

- 이메일 소유 확인 부재
- 비밀번호 분실 복구 부재
- 소셜 로그인 사용자의 비밀번호 설정 부재
- Refresh Token 세션 관리 부재
- 보안 이벤트 기록 부재

## 4. 사용자 가치

사용자는 이메일을 인증하고, 비밀번호를 잊어도 복구할 수 있으며, 자신의 로그인 세션을 확인하고 폐기할 수 있다.

## 5. 구현 범위

- 이메일 인증 발송/재발송/검증
- 비밀번호 찾기
- 비밀번호 재설정
- 로그인 사용자 비밀번호 변경
- 소셜 로그인 사용자 비밀번호 설정
- 세션 목록
- 개별/다른 모든/전체 세션 로그아웃
- 로그인 실패 제한
- 보안 이벤트 기록
- Development outbox
- SMTP Adapter

## 6. 제외 범위

- 이메일 주소 변경
- 회원 탈퇴
- TOTP
- SMS 인증
- WebAuthn
- 실제 운영 SMTP 계정 생성

## 7. 시스템 변경사항

- `account_security.py` endpoint 추가
- email/password reset token 모델 추가
- refresh token 세션 메타데이터 확장
- 보안 설정 화면 추가

## 9. API 변경

| Method | Path | 설명 |
| --- | --- | --- |
| POST | `/api/v1/auth/email-verification/send` | 인증 메일 발송 |
| POST | `/api/v1/auth/email-verification/verify` | 인증 token 검증 |
| POST | `/api/v1/auth/password/forgot` | 비밀번호 재설정 요청 |
| POST | `/api/v1/auth/password/reset` | 비밀번호 재설정 |

| POST | `/api/v1/auth/password/set` | 소셜 계정 비밀번호 설정 |
| GET | `/api/v1/auth/sessions` | 세션 목록 |
| DELETE | `/api/v1/auth/sessions/{sessionId}` | 개별 세션 폐기 |
| DELETE | `/api/v1/auth/sessions/others` | 다른 모든 세션 폐기 |
| DELETE | `/api/v1/auth/sessions` | 전체 세션 폐기 |
| GET | `/api/v1/auth/security-events` | 보안 이벤트 목록 |

# ApplyMate AI v0.1.0

## 1. 버전 개요

v0.1.0은 ApplyMate AI의 프로젝트 기반을 만든 최초 버전이다. 이후 인증, 프로필, 채용공고, AI 분석 기능을 올릴 수 있도록 Frontend, Backend, Database, Docker Compose의 기본 골격을 구축했다.

## 2. 개발 배경

문서와 코드가 분리되지 않은 상태에서 서비스를 확장하면 API, DB, 환경변수, 실행 방식이 빠르게 불명확해진다. 따라서 첫 단계에서는 실제 기능보다 실행 가능한 개발 환경과 공통 구조를 우선했다.

## 3. 해결하려는 문제

- Frontend와 Backend의 독립 실행 구조 부재
- PostgreSQL, Redis 기반 로컬 개발 환경 부재
- API 공통 응답/오류 형식 부재
- 추후 migration과 테스트를 올릴 프로젝트 구조 부재

## 4. 사용자 가치

직접적인 사용자 기능은 제한적이지만, 이후 회원가입과 채용공고 기능을 안정적으로 개발할 수 있는 기반을 제공한다.

## 5. 구현 범위

- Next.js 초기 프로젝트
- FastAPI 초기 프로젝트
- PostgreSQL, Redis Docker Compose 구성
- Backend Health API
- 공통 API 응답 구조
- 초기 README와 문서 폴더 구조

## 6. 제외 범위

- 회원가입/로그인
- 커리어 프로필
- 채용공고 관리
- AI 기능
- 운영 배포 자동화

## 7. 시스템 변경사항

- `frontend/`, `backend/`, `docs/`, `infra/` 기반 구조 생성
- Docker Compose로 frontend/backend/postgres/redis 실행 구조 준비

## 8. Backend 구현

- FastAPI 앱 엔트리포인트
- `/api/v1/health`
- DB/Redis 연결 상태 확인
- 공통 응답 스키마의 출발점

## 9. API 변경

| Method | Path | 인증 | 설명 |
| --- | --- | --- | --- |
| GET | `/api/v1/health` | 불필요 | Backend, PostgreSQL, Redis 상태 확인 |

## 10. Database 변경

초기 버전에서는 운영 테이블을 만들지 않았다. PostgreSQL 컨테이너와 Alembic 기반을 준비했다.

## 11. Frontend 구현

- Next.js App Router 기반 초기 화면
- 서비스 상태 확인 컴포넌트의 기반

## 12. 보안 및 권한

인증 기능 전 단계이므로 보호 API는 없다. 단, secret은 `.env` 기반으로 관리한다는 원칙을 세웠다.

- `NEXT_PUBLIC_API_BASE_URL`

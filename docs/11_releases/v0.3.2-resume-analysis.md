# v0.3.2 AI Resume Analysis

v0.3.2는 v0.3.1에서 추출한 이력서 텍스트를 AI Provider로 구조화 분석하는 단계다.

## Implemented

- 추출 완료된 `raw_text` 또는 사용자 `edited_text` 기반 이력서 분석
- 입력 우선순위: `edited_text` > `raw_text`
- 분석 상태: `PENDING`, `PROCESSING`, `COMPLETED`, `FAILED`, `INVALID_OUTPUT`, `PROVIDER_UNAVAILABLE`
- 구조화 결과: 요약, headline, 기술, 경력, 프로젝트, 학력, 자격증, 성과, 수상, 포트폴리오, 언어, 연락처
- Evidence 검증: 핵심 항목은 원문에 존재하는 `source_text` 근거가 필요하다.
- 사용자 수정본 분리 저장: `structured_result`와 `edited_result` 분리
- 재분석 시 기존 사용자 수정본을 자동 삭제하지 않는다.
- 분석 실행 이력 저장
- 프로필 반영 후보 제공: `ADD`, `UPDATE`, `DUPLICATE`, `CONFLICT`
- Mock Provider와 OpenAI Provider 진입점
- Prompt injection 방어 문구가 포함된 resume analysis prompt

## API

- `POST /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `PATCH /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `DELETE /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `POST /api/v1/resumes/{resumeId}/files/{fileId}/analysis/retry`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/runs`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/runs/{runId}`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/profile-candidates`
- `GET /api/v1/ai/resume-providers`

## Verification notes

- Mock Provider는 결정적 결과를 반환하며 실제 OpenAI 호출 없이 UI/API 흐름을 검증할 수 있다.
- OpenAI Provider 구현 진입점은 추가했지만, 실제 유료 API 호출 검증은 `NEEDS_VERIFICATION`으로 남긴다.

## Limitations

- OCR은 v0.3.2 범위가 아니다.
- 분석 결과는 프로필에 자동 반영하지 않는다.
- 근거가 없는 항목은 저장하지 않고 invalid output으로 처리한다.

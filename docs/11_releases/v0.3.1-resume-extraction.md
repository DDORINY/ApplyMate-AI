# v0.3.1 Resume Text Extraction

## Summary

v0.3.1은 업로드된 PDF/DOCX 이력서 파일에서 텍스트를 추출하고, 최신 결과와 실행 이력을 저장하는 단계다.

## Backend

- PDF/DOCX 텍스트 추출 API 추가
- 사용자 수정 텍스트 저장 API 추가
- 재추출 API 추가
- 추출 실행 이력 조회 API 추가
- `TEXT_NOT_FOUND`, `OCR_REQUIRED` 상태 분리
- 사용자 소유권 검사 적용

## Frontend

- 이력서 상세 화면에서 파일별 텍스트 추출 실행
- 추출 결과, 페이지별 텍스트, 섹션 후보 표시
- 사용자 수정 텍스트 저장
- 재추출 실행
- 실행 이력 표시
- 회원/계정 네비게이션을 우측 계정 영역으로 분리

## DB

- migration head: `20260719_1600`
- `resume_file_extractions`
- `resume_extraction_runs`

## Extraction Policy

- PDF: text literal 기반 best-effort 추출
- DOCX: `word/document.xml` 텍스트 노드 추출
- OCR: v0.3.1 제외
- 텍스트 레이어 없는 PDF: `OCR_REQUIRED`
- 텍스트가 비어 있는 파일: `TEXT_NOT_FOUND`
- DOCX 페이지: 단일 logical page
- 섹션 후보: 규칙 기반 힌트이며 AI 분석 결과가 아님

## Validation

릴리스 전 다음 항목을 검증한다.

- Backend ruff
- Backend pytest
- Frontend lint
- Frontend type-check
- Frontend build
- Docker compose config
- 격리 PostgreSQL migration 왕복 검증
- PDF/DOCX API smoke
- 사용자 수정, 재추출, 실행 이력, 소유권 테스트

## Known Limitations

- OCR은 실행하지 않는다.
- 이미지 기반 PDF는 텍스트 추출 대신 `OCR_REQUIRED`로 표시한다.
- PDF 페이지별 텍스트는 best-effort다.
- DOCX의 실제 렌더링 페이지 경계는 제공하지 않는다.

# v0.2.2 적합도 점수 산식

## 점수 구성

| 영역 | 가중치 | 설명 |
| --- | ---: | --- |
| 직무 적합도 | 25% | 희망 직무, headline, 경력/프로젝트 role과 공고 직무 키워드 비교 |
| 기술 적합도 | 30% | 공고 분석에서 추출된 기술과 사용자 기술 비교 |
| 경력 적합도 | 15% | 공고 분석의 최소 경력 요구와 사용자 경력 연차 비교 |
| 프로젝트 적합도 | 15% | 프로젝트 기술/역할/설명과 공고 기술·업무 키워드 비교 |
| 희망 조건 | 10% | 고용형태, 근무형태, 지역, 급여 조건 비교 |
| 위험/제외 조건 | 5% | 지원 제외 조건과 필수 기술 누락 위험 반영 |

종합 점수는 0~100 범위로 clamp합니다.

## 등급

| 점수 | 등급 |
| ---: | --- |
| 85~100 | `EXCELLENT` |
| 70~84 | `GOOD` |
| 50~69 | `MODERATE` |
| 30~49 | `LOW` |
| 0~29 | `VERY_LOW` |

## 추천 상태

| 조건 | 상태 |
| --- | --- |
| 프로필 완성도 35 미만 | `INSUFFICIENT_DATA` |
| 활성 지원 제외 조건과 일치 | `NOT_RECOMMENDED` |
| 점수 85 이상 | `STRONGLY_RECOMMENDED` |
| 점수 70 이상 | `RECOMMENDED` |
| 점수 50 이상 | `CONSIDER` |
| 그 외 | `NOT_RECOMMENDED` |

## 최신성 판단

다음 hash가 저장 시점과 현재 시점에 다르면 `is_outdated=true`입니다.

- `profile_hash`: 프로필, 기술, 경력, 프로젝트, 희망 조건, 활성 제외 조건
- `job_analysis_hash`: 완료된 공고 분석 결과와 입력 hash

## AI 사용 범위

v0.2.2에서는 점수 계산이 모두 규칙 기반입니다. `AI_PROVIDER=disabled`인 경우에도 template 설명으로 정상 동작합니다.

# 향후 상세 개발 계획

## v0.3.0 이력서 파일 업로드

- 목적: 사용자가 PDF/DOCX 이력서를 업로드한다.
- 사용자 가치: 기존 이력서를 서비스에 연결한다.
- 구현 범위: 파일 업로드, 파일 크기/확장자/MIME 검증, 저장 메타데이터.
- 제외 범위: AI 분석, 문서 생성.
- 예상 DB: `resumes`, `resume_files`.
- 예상 API: `POST /resumes`, `GET /resumes`, `DELETE /resumes/{resumeId}`.
- 예상 Frontend: `/resumes`.
- 보안: 파일명 직접 사용 금지, 크기 제한, MIME 검증.
- 테스트: 업로드/삭제/소유권/잘못된 파일.
- 선행 조건: 파일 저장 위치 결정.
- 완료 조건: 안전한 파일 업로드와 조회.

## v0.3.1 이력서 텍스트 추출

- 목적: 업로드 파일에서 텍스트를 추출한다.
- 구현 범위: PDF/DOCX parser, 추출 결과 저장, 실패 이력.
- 제외 범위: AI 구조화 분석.
- 예상 DB: `resume_extractions`.
- 보안: parser 오류 격리, 파일 크기 제한 유지.

## v0.3.2 AI 이력서 구조화 분석

- 목적: 이력서 텍스트를 경력/프로젝트/기술/성과로 구조화한다.
- 구현 범위: AI Provider 사용, schema 검증, 사용자 검토/수정.
- 제외 범위: 자동으로 프로필 덮어쓰기.
- 보안: 없는 경력 생성 금지.

## v0.3.3 맞춤 지원 문서 생성

- 목적: 공고 분석, 적합도 분석, 사용자 프로필을 근거로 지원 문서 초안을 만든다.
- 구현 범위: 자기소개서/지원동기 초안, 근거 표시, 사용자 확인 필요 표시.
- 제외 범위: 합격 가능성 확정 표현.
- 보안: 실제 경험에 없는 성과 생성 금지.

## v0.4.0 지원 현황 관리

- 목적: 공고별 지원 상태를 관리한다.
- 예상 DB: `applications`.
- 예상 상태: SAVED, PREPARING, APPLIED, INTERVIEW, OFFER, REJECTED, WITHDRAWN.

## v0.4.1 일정 관리

- 목적: 마감일, 면접일, 과제 제출일을 관리한다.
- 예상 DB: `calendar_events`.
- 일정 신뢰도: CONFIRMED, ESTIMATED, USER_INPUT.

## v0.4.2 대시보드

- 목적: 지원 현황, 마감 임박, 추천 작업을 한 화면에서 보여준다.

## v0.5.0 Google Calendar 연동

- 목적: 내부 일정을 외부 캘린더와 동기화한다.
- 선행 조건: Google OAuth scope 재설계, provider token 저장 정책 수립.

## v0.5.1 Gmail 채용 메일 분석

- 목적: 채용 관련 이메일에서 일정/공고/상태를 추출한다.
- 선행 조건: Gmail scope, 개인정보 정책, 사용자 명시 승인.

## v0.6.0 추천 시스템 기반

- 목적: 저장된 공고와 프로필로 추천 후보를 구성한다.
- 예상 기술: 필터 + 점수 기반 정렬, 이후 embedding 검토.

## v0.6.1 정기 추천

- 목적: 사용자에게 정기적으로 추천 공고를 제공한다.
- 선행 조건: 추천 pool, background job, 알림 정책.

## v1.0.0 MVP

- 목적: 프로필→공고→분석→적합도→지원 문서→지원 현황→일정 흐름을 통합한다.
- 완료 조건: 주요 사용자 흐름 E2E, 보안 점검, 운영 배포 문서, 백업/모니터링 계획.

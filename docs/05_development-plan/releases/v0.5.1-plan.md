# ApplyMate AI v0.5.1 Gmail 분석 계획

## 1. 버전 목적

v0.5.1은 사용자가 명시적으로 연결한 Gmail 계정에서 취업 활동과 관련된 메일을 분석하여 지원 현황과 일정 관리의 입력 후보를 만드는 단계이다.

이 버전은 메일 내용을 자동으로 변경하거나 확정 반영하지 않는다. ApplyMate AI는 분석 결과를 후보로 제시하고, 사용자가 확인한 항목만 내부 데이터로 저장한다.

## 2. 사용자 가치

- 채용 관련 메일을 직접 뒤지지 않아도 지원 결과, 면접 일정, 과제 안내 후보를 확인할 수 있다.
- Gmail에서 발견한 정보를 기존 지원 현황과 일정 관리 기능에 연결할 수 있다.
- 자동 처리보다 사용자 확인을 우선하여 개인정보와 일정 신뢰도를 지킨다.

## 3. 선행 조건

- v0.5.0 Google Calendar 연동 기반이 main에 병합되어 있어야 한다.
- 외부 OAuth 계정, 토큰 암호화, 연결 상태, 동기화 오류 기록 구조를 재사용한다.
- Gmail OAuth는 로그인 OAuth 및 Calendar OAuth와 분리한다.

## 4. 구현 범위

- Gmail OAuth 연결 기반
- Gmail Provider 인터페이스
- Mock Gmail Provider
- Google Gmail Provider 골격
- 채용 관련 메일 검색 조건 구조화
- 메일 분석 후보 생성
- 지원 결과 후보
- 면접 일정 후보
- 과제 일정 후보
- 지원 항목 연결 후보
- 사용자 확인 후 내부 반영 API 설계
- 프론트 외부 연동 화면에 Gmail 섹션 추가
- 문서 및 테스트 최신화

## 5. 제외 범위

- 사용자의 승인 없는 Gmail 메일 수정, 삭제, 발송
- 사용자의 승인 없는 지원 상태 변경
- 사용자의 승인 없는 일정 생성
- 메일 전체함 대규모 크롤링
- 광고성/비채용 메일의 일반 요약 기능
- 실제 Gmail API 운영 검증 완료 선언

실제 Gmail API 호출은 v0.5.1에서 `NEEDS_VERIFICATION`으로 관리한다.

## 6. OAuth 정책

Gmail OAuth는 다음 원칙을 따른다.

- 로그인 OAuth와 Gmail OAuth를 분리한다.
- Calendar OAuth와 Gmail OAuth도 목적을 분리한다.
- Gmail 연결 전 사용 목적과 접근 범위를 화면에 설명한다.
- OAuth state를 검증하고 1회 사용 후 재사용을 차단한다.
- 토큰은 평문으로 저장하지 않는다.
- 연결 해제 시 내부 분석 후보는 보존할 수 있으나 Gmail 토큰은 폐기한다.

## 7. 권장 Gmail Scope

초기 구현은 읽기 중심 권한을 우선한다.

```text
https://www.googleapis.com/auth/gmail.readonly
```

향후 메일 라벨링이나 보관 처리가 필요하면 별도 버전에서 권한 확대를 검토한다.

## 8. 데이터 모델 초안

기존 `external_accounts` 구조를 재사용하되 purpose를 Gmail 분석용으로 확장한다.

신규 테이블 후보:

- `gmail_analysis_requests`
- `gmail_message_candidates`
- `gmail_application_candidates`
- `gmail_schedule_candidates`
- `gmail_analysis_errors`

주요 필드:

- `user_id`
- `external_account_id`
- `gmail_message_id`
- `thread_id`
- `subject`
- `sender`
- `received_at`
- `snippet`
- `candidate_type`
- `confidence`
- `evidence`
- `status`
- `confirmed_at`
- `dismissed_at`

## 9. 분석 후보 유형

### 지원 결과 후보

- 서류 합격
- 서류 불합격
- 면접 합격
- 최종 합격
- 불합격
- 추가 확인 필요

### 일정 후보

- 면접 일정
- 과제 제출 마감
- 코딩 테스트 일정
- 설명회 또는 채용 행사
- 서류 제출 마감

### 연결 후보

- 기존 지원 항목과 연결 가능
- 새 지원 항목 생성 후보
- 기존 채용공고와 연결 가능
- 사용자 직접 확인 필요

## 10. 사용자 확인 정책

다음 작업은 반드시 사용자 확인 후 수행한다.

- Gmail 분석 후보를 내부 지원 현황에 반영
- Gmail 분석 후보를 내부 일정으로 생성
- 기존 지원 상태 변경
- 기존 일정 변경
- Google Calendar로 일정 동기화
- Gmail 메일에 라벨을 붙이거나 읽음 처리하는 향후 기능

## 11. API 초안

```text
GET    /api/v1/integrations/gmail/status
POST   /api/v1/integrations/gmail/connect
GET    /api/v1/integrations/gmail/callback
DELETE /api/v1/integrations/gmail/connection

POST   /api/v1/gmail/analysis/requests
GET    /api/v1/gmail/analysis/requests
GET    /api/v1/gmail/analysis/candidates
POST   /api/v1/gmail/analysis/candidates/{candidate_id}/confirm
POST   /api/v1/gmail/analysis/candidates/{candidate_id}/dismiss
```

## 15. 리스크와 보류 항목

- Gmail API 실제 호출은 Google Cloud OAuth 설정 후 별도 검증이 필요하다.
- 메일 본문에는 민감정보가 포함될 수 있으므로 로그에 원문을 남기지 않는다.
- AI 분석이 확정 사실처럼 보이지 않도록 후보/근거/신뢰도를 함께 표시한다.
- 사용자가 확인하지 않은 후보는 지원 상태나 일정으로 자동 반영하지 않는다.

# v0.8.0 알림·리마인더 운영화 상세 개발계획

## 1. 목적

기존 일정 리마인더, 추천 알림 후보, Gmail 후보, 문서 개선 실행 결과, 동기화 실패 이벤트를 하나의 사용자 알림 구조로 통합한다.

## 3. 구현 범위

- In-app 알림 저장/목록/상세/읽음/전체 읽음/해제/보관
- 사용자별 알림 설정

## 7. In-app 알림

목록, 읽지 않은 개수, 상세 이동, 개별 읽음, 전체 읽음, 해제, 보관, 필터, 페이지네이션을 제공한다.

## 8. 일정 리마인더

`schedule_reminders.status=ACTIVE`, `scheduled_at <= now`, 일정이 취소/보관되지 않은 항목을 처리한다. 처리 후 reminder는 `SENT` 또는 `FAILED`로 변경한다.

## 9. 추천 알림 후보

`recommendation_notification_candidates.status=PENDING` 후보를 notification으로 변환한다. 기존 후보는 삭제하지 않고 처리 상태로 변경한다.

## 10. 지원 상태 알림

지원 상태 변경 이력을 source로 알림을 만들 수 있는 구조를 둔다. 자동 지원 상태 변경은 하지 않는다.

## 11. Gmail 후보 알림

`email_candidates.status=NEW` 후보를 `GMAIL_CANDIDATE_CREATED` 알림으로 생성한다. 알림은 후보 상세 또는 후보 목록으로 연결한다.

## 12. 문서 개선 알림

`document_improvement_runs`의 `COMPLETED`, `REVIEW_REQUIRED`, `APPLIED`, `FAILED` 상태를 알림으로 연결한다. provider 원문 오류와 prompt는 알림에 포함하지 않는다.

## 13. Background Worker

별도 worker process에서 task를 실행할 수 있게 한다. v0.8.0은 무거운 queue 도입 대신 idempotent task 함수와 Docker Compose worker service를 추가한다.

## 14. 재시도 정책

Delivery는 최대 3회까지 재시도한다. 기본 backoff는 1분, 5분, 30분이다.

## 15. 중복 방지

`deduplication_key`에 unique constraint를 둔다. 코드 확인만으로 중복을 막지 않는다.

## 16. 이메일 Provider

- `DisabledNotificationEmailProvider`
- `MockNotificationEmailProvider`
- `SmtpNotificationEmailProvider`

실제 SMTP 발송은 운영 credential 검증 전까지 `NEEDS_VERIFICATION`이다.

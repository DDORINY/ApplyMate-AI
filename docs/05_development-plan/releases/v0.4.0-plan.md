# v0.4.0 지원 현황 관리 상세 개발계획

## 1. 버전 목적

채용공고를 실제 지원 항목으로 전환하고, 지원 상태·상태 변경 이력·메모·연결 문서를 관리한다.

## 2. 사용자 가치

사용자는 어떤 공고에 지원 준비 중인지, 제출했는지, 면접/과제/합격/불합격 상태인지 한 곳에서 추적할 수 있다.

## 3. 선행 조건

- v0.3.3 맞춤 지원 문서 생성 완료
- `job_postings`, `application_documents` 존재
- migration head `20260719_1800`

## 4. 구현 범위

- 지원 항목 생성/목록/상세/수정/보관
- 지원 상태 변경
- 상태 변경 이력 저장
- 지원 메모 저장
- 지원 문서 연결
- 검색, 필터, 정렬, 페이지네이션
- 프론트 `/applications` 화면

## 5. 제외 범위

- 외부 채용 사이트 자동 제출
- Google Calendar 실제 일정 생성
- 메일 자동 발송
- 합격 가능성 예측

## 11. API 초안

- `POST /api/v1/applications`
- `GET /api/v1/applications`
- `GET /api/v1/applications/{applicationId}`
- `PATCH /api/v1/applications/{applicationId}`
- `DELETE /api/v1/applications/{applicationId}`
- `POST /api/v1/applications/{applicationId}/status`
- `GET /api/v1/applications/{applicationId}/status-history`
- `POST /api/v1/applications/{applicationId}/notes`
- `GET /api/v1/applications/{applicationId}/notes`
- `PATCH /api/v1/applications/{applicationId}/notes/{noteId}`
- `DELETE /api/v1/applications/{applicationId}/notes/{noteId}`

## 12. Frontend 초안

- `/applications`: 목록, 상태 필터, 검색, 정렬
- `/applications/new`: 공고 기반 지원 항목 생성
- `/applications/{applicationId}`: 상태 변경, 연결 문서, 메모, 이력

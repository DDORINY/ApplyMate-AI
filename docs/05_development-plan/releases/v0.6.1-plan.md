# v0.6.1 추천 UX 개선 및 추천 실행 자동화 기반

## 1. 목적

v0.6.1은 v0.6.0의 수동 규칙 기반 추천을 확장해 사용자별 추천 실행 설정, 스케줄러가 호출 가능한 실행 조건 판단, 추천 Snapshot, 신규·변경 추천 판정, 알림 후보 저장, 추천 목록 UX 개선 기반을 제공한다.

## 2. 현재 문제

- 추천 생성은 사용자가 직접 버튼을 눌러야 한다.
- 추천 결과가 언제 기준으로 생성되었는지 묶어서 보기 어렵다.
- 이전 추천 대비 새 추천, 점수 상승·하락, 등급 변화가 구조화되어 있지 않다.
- 숨김/관심 없음/나중에 보기 등 피드백이 목록 UX와 자동 실행 정책에 충분히 반영되지 않는다.
- 실제 이메일·푸시 발송 전 단계인 알림 후보 저장 구조가 없다.

## 3. 구현 범위

- 사용자별 추천 실행 설정
- `MANUAL`, `DAILY`, `WEEKLY` 실행 정책
- 스케줄러/향후 Worker가 호출 가능한 `run-if-due` API
- 추천 Snapshot과 Snapshot item 저장
- 신규·변경·제거 추천 판정
- 점수/등급/순위 변화량 저장
- 추천 알림 후보 저장과 읽음/해제 처리
- `/recommendations` 목록 UX 개선
- `/recommendations/history` 추천 이력 화면
- `/settings/recommendations` 추천 설정 화면
- API/DB/문서/테스트 최신화

## 4. 제외 범위

- 외부 채용공고 자동 수집
- 실제 이메일 발송
- 실제 푸시 알림 발송
- AI/ML 기반 추천 모델
- 사용자 승인 없는 지원 상태 변경
- 자동 지원 제출
- 실제 Background Worker 운영 완료 표시

## 5. 추천 실행 정책

v0.6.1의 실행 정책은 추천 생성 여부를 판단하고, 실행 가능한 경우 기존 v0.6.0 추천 Service를 호출한다. 추천 계산 로직은 중복 구현하지 않는다.

실행하지 않는 사유:

- `DISABLED`
- `NOT_DUE`
- `PROFILE_MISSING`
- `NO_ACTIVE_JOBS`
- `DUPLICATE_INPUT`
- `ALREADY_RUNNING`

## 6. 자동 실행 설정

사용자별 설정 필드:

- `enabled`
- `frequency`
- `preferred_run_hour`
- `timezone`
- `minimum_score`
- `include_jobs_without_analysis`
- `exclude_applied_jobs`
- `exclude_hidden_jobs`
- `notify_new_recommendations`
- `notify_score_changes`
- `last_run_at`
- `next_run_at`

기본값은 `enabled=false`, `frequency=MANUAL`이다. 사용자가 명시적으로 활성화해야 자동 실행 후보가 된다.

## 7. 추천 Snapshot

추천 실행마다 Snapshot을 저장한다.

- `run_id`
- `user_id`
- `generated_at`
- `profile_hash`
- `policy_version`
- `input_job_count`
- `recommended_count`
- `new_count`
- `changed_count`
- `removed_count`

Snapshot item은 추천별 점수, 등급, 순위, 변화 판정, 이전 값과 변화량을 저장한다.

## 8. 신규·변경 추천 판정

이전 최신 Snapshot과 현재 추천 결과를 비교한다.

- `NEW`
- `UNCHANGED`
- `SCORE_UP`
- `SCORE_DOWN`
- `GRADE_UP`
- `GRADE_DOWN`
- `REMOVED`
- `OUTDATED`

Frontend는 점수 차이를 직접 계산하지 않고 Backend 판정값을 표시한다.

## 9. 피드백 반영

v0.6.1은 v0.6.0 피드백을 필터, 정렬, 제외 정책에 반영한다.

- 숨김 공고 기본 제외
- 지원 완료 공고 기본 제외
- 관심 있음 공고 상단 표시 가능
- 나중에 보기 필터 제공
- 관심 없음 사유별 필터 제공

피드백을 기반으로 추천 가중치를 자동 학습하지 않는다.

## 10. 목록 UX

`/recommendations`는 다음 배지를 표시한다.

- 새 추천
- 점수 상승
- 점수 하락
- 관심 있음
- 나중에 보기
- 숨김
- 지원 완료
- 오래된 추천

필터:

- `change_type`
- `feedback_type`
- `grade`
- `minimum_score`
- `has_blocking_mismatch`
- `outdated`
- `keyword`

정렬:

- `score`
- `score_delta`
- `rank`
- `rank_delta`
- `generated_at`
- `deadline`

## 11. 빈 상태와 데이터 부족 상태

빈 상태를 다음처럼 구분한다.

- 프로필 없음
- 희망 직무 없음
- 기술 정보 없음
- 저장된 공고 없음
- 활성 공고 없음
- 추천 조건 충족 공고 없음
- 모든 공고가 지원 완료
- 모든 공고가 숨김
- 추천 실행 전
- 추천 실행 실패

각 상태는 프로필 작성, 기술 추가, 공고 등록, 숨김 목록 보기, 추천 실행 CTA로 연결한다.

## 12. 알림 후보

실제 발송 대신 알림 후보만 저장한다.

유형:

- `NEW_HIGH_SCORE_RECOMMENDATION`
- `RECOMMENDATION_SCORE_INCREASED`
- `RECOMMENDATION_GRADE_INCREASED`
- `APPLICATION_DEADLINE_APPROACHING`
- `RECOMMENDATION_BECAME_OUTDATED`

상태:

- `PENDING`
- `READ`
- `DISMISSED`
- `EXPIRED`

## 13. DB 설계

신규 테이블:

- `job_recommendation_settings`
- `job_recommendation_snapshots`
- `job_recommendation_snapshot_items`
- `recommendation_notification_candidates`

모든 사용자 데이터는 `user_id` 소유권을 갖고, Snapshot item은 Snapshot에 종속된다.

## 14. API 설계

- `GET /api/v1/recommendations/settings`
- `PATCH /api/v1/recommendations/settings`
- `POST /api/v1/recommendations/jobs/run-if-due`
- `GET /api/v1/recommendations/jobs/snapshots`
- `GET /api/v1/recommendations/jobs/snapshots/{snapshotId}`
- `GET /api/v1/recommendations/jobs/changes`
- `GET /api/v1/recommendation-notifications`
- `PATCH /api/v1/recommendation-notifications/{notificationId}`
- `DELETE /api/v1/recommendation-notifications/{notificationId}`

`run-if-due`는 인증된 사용자 또는 향후 내부 스케줄러가 호출 가능한 구조로 설계하며 공개 endpoint로 만들지 않는다.

## 16. Frontend 설계

추가·개선 화면:

- `/recommendations`
- `/recommendations/history`
- `/settings/recommendations`

대시보드는 저장된 Snapshot만 조회하고 자동 실행을 수행하지 않는다.

## 17. 보안

- 모든 설정, Snapshot, 알림 후보는 `current_user.id`로 제한한다.
- 알림 후보 payload에는 secret, token, 원문 이메일, 민감한 provider 응답을 저장하지 않는다.
- 외부 크롤링과 실제 알림 발송을 수행하지 않는다.
- 중복 실행 방지를 위해 실행 중 run과 동일 입력 hash를 확인한다.

## 18. 테스트

Backend:

- 설정 기본값/수정/소유권
- 실행 조건 `DISABLED`, `NOT_DUE`, `PROFILE_MISSING`, `NO_ACTIVE_JOBS`, `ALREADY_RUNNING`
- Snapshot 최초 생성, 신규 추천, 점수 변화, 등급 변화
- 알림 후보 생성, 읽음, 해제, 소유권
- 피드백 기반 숨김 제외

Frontend:

- 추천 목록 필터/배지
- 추천 설정 폼
- 추천 이력 화면
- 알림 후보 목록
- 빈 상태/오류 상태

## 19. Migration

신규 migration은 현재 head `20260719_2300` 다음 revision으로 작성한다.

목표 파일:

```text
20260720_0000_create_recommendation_automation_tables.py
```

독립 Docker project에서 upgrade/downgrade 왕복을 검증한다. 기존 Docker volume은 삭제하지 않는다.

# v0.6.1 일일 맞춤 추천 및 추천 이력 상세 개발계획

## 1. 버전 목적

v0.6.1은 v0.6.0의 수동 규칙 기반 추천을 확장해 사용자별 추천 실행 설정, 추천 snapshot, 추천 변화 감지, 알림 후보 생성 기반을 마련한다.

## 2. 사용자 가치

사용자는 매번 직접 추천을 생성하지 않아도 새로 저장된 공고와 변경된 프로필 기준으로 추천 변화를 확인할 수 있다. 추천이 왜 새로 등장했는지, 점수가 왜 바뀌었는지도 추적할 수 있다.

## 3. 선행 조건

- v0.6.0 규칙 기반 채용공고 추천
- 추천 실행 이력
- 추천 결과/이유/피드백 테이블
- 일정 및 알림 저장 정책

## 4. 구현 범위

- 사용자별 추천 실행 설정
- 추천 배치 실행 구조
- 일일 추천 snapshot
- 추천 이력 조회
- 새 추천 감지
- 추천 점수/등급 변화 기록
- 알림 후보 생성
- 문서, API, DB, 테스트 최신화

## 5. 제외 범위

- 외부 채용공고 자동 수집
- 실제 이메일 발송
- 실제 푸시 알림 발송
- AI/ML 기반 추천 모델
- 사용자 승인 없는 지원 상태 변경
- 자동 지원 제출

## 6. 사용자 흐름

추천 설정 확인 → 일일 추천 실행 → 추천 snapshot 생성 → 새 추천/점수 변화 확인 → 알림 후보 검토 → 추천 상세 또는 공고 상세로 이동.

## 7. 추천 실행 설정

사용자별로 다음 설정을 저장한다.

- 추천 자동 실행 사용 여부
- 실행 주기
- 기준 시간대
- 기본 후보 수
- 이미 지원한 공고 제외 여부
- 숨김/관심 없음 피드백 제외 여부
- 데이터 부족 추천 포함 여부

## 8. 추천 배치 실행

초기 구현은 운영 스케줄러와 분리된 내부 service 단위로 작성한다.

- 단일 사용자 실행
- 전체 사용자 실행 후보 조회
- 실행 중복 방지
- 실패 이력 저장
- 재시도 정책은 문서화하되 과도하게 구현하지 않는다.

## 9. 일일 추천 snapshot

추천 snapshot은 특정 날짜/사용자/정책 버전 기준 추천 결과 묶음이다.

저장 후보:

- snapshot date
- user_id
- run_id
- policy_version
- total_count
- new_count
- changed_count
- top_score
- generated_at

## 10. 추천 변화 감지

동일 사용자와 공고 기준으로 이전 snapshot과 비교한다.

- 신규 추천
- 점수 상승
- 점수 하락
- 등급 변경
- 필수 조건 불일치 변경
- 오래된 추천 갱신

## 11. 알림 후보

v0.6.1은 실제 발송이 아니라 알림 후보 저장까지만 구현한다.

알림 후보 예시:

- 새 추천 공고
- 점수 상승 공고
- 마감 임박 추천 공고
- 프로필 보완 필요 추천

## 12. API 초안

- `GET /api/v1/recommendations/jobs/settings`
- `PATCH /api/v1/recommendations/jobs/settings`
- `POST /api/v1/recommendations/jobs/batch/run`
- `GET /api/v1/recommendations/jobs/snapshots`
- `GET /api/v1/recommendations/jobs/snapshots/{snapshotId}`
- `GET /api/v1/recommendations/jobs/changes`
- `GET /api/v1/recommendations/jobs/notification-candidates`

최종 API는 구현 전 `docs/04_api/api-specification.md`에 확정한다.

## 13. DB 초안

후보 테이블:

- `job_recommendation_settings`
- `job_recommendation_snapshots`
- `job_recommendation_snapshot_items`
- `job_recommendation_changes`
- `job_recommendation_notification_candidates`

DB 변경 시 Alembic migration을 추가하고 upgrade/downgrade를 검증한다.

## 14. Frontend 초안

- 추천 설정 UI
- 추천 snapshot 목록
- 추천 변화 배지
- 새 추천/점수 상승/마감 임박 필터
- 알림 후보 목록

## 15. 정책

- 추천 계산 자체는 v0.6.0의 `RULE_BASED` 정책을 재사용한다.
- 자동 실행은 추천 생성만 수행하며 지원 상태를 변경하지 않는다.
- 알림 후보는 사용자에게 보여줄 후보이며 실제 발송 완료로 표시하지 않는다.
- 외부 채용공고 수집은 포함하지 않는다.

## 17. 완료 기준

- v0.6.1 문서와 코드가 일치한다.
- 실제 이메일/푸시 발송을 완료로 표시하지 않는다.
- 외부 채용공고 자동 수집을 구현하지 않는다.
- 추천 변화와 snapshot이 사용자 소유권 기준으로 제한된다.
- 테스트와 build가 통과한다.

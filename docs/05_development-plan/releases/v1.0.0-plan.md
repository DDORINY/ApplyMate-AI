# v1.0.0 MVP 릴리스 상세 계획

## 1. 릴리스 목적

ApplyMate AI v1.0.0은 개인용 AI 취업 매니저 MVP 릴리스다. v0.9.0까지 구현된 기능을 기준으로 사용자 흐름, 운영 설정, 문서, 테스트, 보안 점검, 릴리스 태그를 마무리한다.

## 2. MVP 확정 범위

- 회원가입·로그인
- 커리어 프로필
- 채용공고 등록·관리
- 채용공고 분석
- 사용자-공고 적합도 분석
- 이력서 업로드·추출·분석
- 지원 문서 생성
- 지원 문서 개선
- 지원 현황 관리
- 일정 관리
- 대시보드
- Google Calendar 연동 기반
- Gmail 채용 메일 분석 기반
- 규칙 기반 채용공고 추천
- 추천 이력·설정
- In-app 알림
- 알림 설정
- Worker 기반 리마인더
- 감사 로그
- 보안·Rate Limit·Health Check

## 3. 제외 범위

- 외부 채용 사이트 자동 크롤링
- 자동 지원 제출
- Push 실발송
- 실제 SMTP 미검증 발송
- 실제 Google OAuth/Gmail/Calendar API 미검증 동작
- 실제 OpenAI 미검증 호출
- 다중 서버 분산 Worker
- Redis 기반 분산 Rate Limit
- 운영 수준 자동 확장
- 모바일 네이티브 앱

## 4. 핵심 사용자 흐름

- 기본 취업 준비: 회원가입 → 로그인 → 프로필 → 공고 → 분석 → 적합도 → 추천
- 이력서와 지원 문서: 업로드 → 추출 → 분석 → 문서 생성 → 개선 → 승인 → 새 버전
- 지원 관리: 지원 항목 → 상태 변경 → 일정 → 대시보드 → 알림
- 외부 연동 Mock: Calendar mock → Gmail mock → 후보 승인
- 알림: 리마인더 → worker → in-app 알림 → 읽음/해제

## 5. 기능 인수 기준

각 기능은 `IMPLEMENTED`, `VERIFIED_MOCK`, `VERIFIED_LOCAL`, `VERIFIED_E2E`, `NEEDS_VERIFICATION`, `NOT_IMPLEMENTED`, `EXCLUDED` 중 하나로 상태를 기록한다.

## 6. E2E 인수 테스트

Playwright smoke를 10개 이상으로 확장한다. 실제 외부 provider는 호출하지 않는다.

## 7. 실제 Provider 검증 범위

v1.0.0에서 mock/local 검증과 실제 provider 검증을 분리한다. OpenAI, Google Calendar, Gmail, SMTP는 운영 credential과 consent가 필요하면 `NEEDS_VERIFICATION`으로 유지한다.

## 8. 운영 환경변수

`.env.example`, `.env.development.example`, `.env.e2e.example`, `.env.production.example`를 분리한다. Secret 기본값은 제공하지 않는다.

## 9. 배포 구조

MVP 권장 배포 구조는 Nginx, Frontend, Backend, Worker, PostgreSQL, Redis이다. 실제 운영 배포를 하지 않으면 배포 완료로 표시하지 않는다.

## 10. 데이터 초기화

재현 가능한 demo seed script를 제공한다. 실제 개인정보와 실제 OAuth token은 포함하지 않는다.

## 11. 데모 계정과 데모 데이터

데모 계정 이메일은 문서화할 수 있지만 비밀번호는 환경변수 또는 실행 인자로 받는다. 실제 비밀번호를 저장소에 기록하지 않는다.

## 12. 사용자 가이드

`docs/09_guides/user-guide.md`에 화면 경로와 기본 사용 흐름을 정리한다.

## 13. 관리자·운영 가이드

`docs/09_guides/operations-guide.md`에 시작·중지, migration, worker, health, backup, restore, provider 설정, 장애 대응을 정리한다.

## 14. 보안 최종 점검

`docs/12_audits/v1.0.0-security-review.md`에 IDOR, XSS, 파일 업로드, prompt injection, OAuth, CORS, rate limit, secret scan, worker/idempotency 결과를 기록한다.

## 15. 성능 최종 점검

성능 smoke script와 결과 문서를 제공한다. 실제 수치를 측정하지 않은 항목은 완료로 표시하지 않는다.

## 16. 백업·복구

운영 절차는 문서화하고, 실제 복구 리허설 여부는 별도 상태로 기록한다.

## 17. 배포 Smoke Test

`scripts/smoke_test.py`로 health/live, health/ready, OpenAPI 문서 endpoint를 점검한다.

## 18. Rollback

이미지/커밋 rollback, migration downgrade 가능성, DB restore 조건을 문서화한다.

## 19. Known Limitations

`docs/00_status/known-limitations.md`에 실제 provider, Push, 분산 worker/rate limit, 운영 배포 미검증 항목을 명시한다.

# v0.3.0 상세 개발계획

## 1. 버전 목적

사용자가 PDF 또는 DOCX 이력서 파일을 안전하게 업로드하고, 파일 메타데이터를 관리하며, 다운로드와 삭제를 수행할 수 있게 한다.

## 2. 사용자 가치

- 지원 준비에 필요한 이력서를 ApplyMate AI에 보관할 수 있다.
- 이후 텍스트 추출과 AI 분석의 근거 파일로 사용할 수 있다.
- 기본 이력서를 지정해 후속 기능에서 우선 참조할 수 있다.

## 3. 선행 조건

- 회원가입/로그인 기능
- 사용자 소유 데이터 권한 검사
- Docker Compose 기반 backend/frontend/database 구조

## 4. 구현 범위

## 6. 사용자 흐름

1. 사용자가 `/resumes`에서 이력서 목록을 확인한다.
2. `/resumes/new`에서 제목과 파일을 입력한다.
3. 업로드가 성공하면 `/resumes/{resumeId}` 상세 화면으로 이동한다.
4. 상세 화면에서 메타데이터, 파일 정보, 다운로드 버튼을 확인한다.
5. 필요하면 기본 이력서로 지정하거나 삭제한다.

## 7. 기능 요구사항

- 로그인한 사용자만 접근 가능해야 한다.
- 모든 조회/수정/삭제는 `user_id` 소유권을 확인해야 한다.
- 허용 확장자는 `.pdf`, `.docx`이다.
- 허용 MIME은 PDF와 Office Open XML 문서 MIME이다.
- 내부 저장명은 UUID 기반이어야 한다.
- 원본 파일명은 표시용 메타데이터로만 저장한다.
- 같은 사용자가 동일 해시 파일을 중복 업로드하면 충돌 오류를 반환한다.
- 기본 이력서는 사용자당 하나만 유지한다.

## 8. 비기능 요구사항

- 파일 업로드 실패 시 DB와 파일 저장 상태가 불일치하지 않아야 한다.
- 다운로드 응답은 `Content-Disposition`을 안전하게 설정해야 한다.
- 삭제는 DB 레코드 삭제 후 파일 삭제를 시도하되, 파일이 이미 없어도 API는 실패하지 않는다.

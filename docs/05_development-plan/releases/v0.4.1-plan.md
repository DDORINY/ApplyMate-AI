# v0.4.1 일정 관리 및 리마인더 상세 개발 계획

## 1. 버전 목적

v0.4.0의 지원 현황을 기반으로 마감일, 지원 예정일, 코딩 테스트, 과제, 면접, 결과 발표, 사용자가 직접 입력한 일정을 관리한다.

## 2. 사용자 가치

사용자는 지원 항목별 중요한 일정을 놓치지 않고 월/주/목록 화면에서 확인하며, 리마인더를 통해 준비해야 할 일을 미리 알 수 있다.

## 3. 선행 조건

- v0.4.0 지원 현황 관리 완료
- `applications` 테이블 존재
- Alembic head: `20260719_1900`

## 4. 구현 범위

- 일정 이벤트 생성/목록/상세/수정/삭제 또는 보관
- 지원 항목과 일정 연결
- 일정 유형 관리: 마감일, 지원 예정일, 면접, 코딩 테스트, 과제, 결과 발표, 사용자 일정
- 일정 신뢰도 관리: `CONFIRMED`, `ESTIMATED`, `USER_INPUT`
- 리마인더 설정 및 조회
- 월/주/목록 기반 프론트 화면
- 지원 상세 화면의 관련 일정 섹션

## 5. 제외 범위

- Google Calendar 실제 일정 생성
- Gmail/이메일 자동 수집
- 푸시 알림/문자 발송
- 외부 캘린더 양방향 동기화

## 6. 사용자 흐름

1. 사용자가 지원 항목 상세에서 일정을 추가한다.
2. 일정 유형과 시작/종료 시각, 신뢰도, 리마인더를 입력한다.
3. 사용자는 월/주/목록 화면에서 모든 지원 일정을 확인한다.
4. 완료되었거나 불필요한 일정은 보관한다.

# v0.3.1 이력서 텍스트 추출 계획

## 목표

v0.3.0에서 업로드한 PDF/DOCX 이력서 파일에서 텍스트를 추출하고, 이후 AI 이력서 구조화 분석의 근거 데이터로 사용할 수 있도록 최신 결과와 실행 이력을 저장한다.

## 구현 범위

- PDF/DOCX 텍스트 추출 실행
- 파일별 최신 추출 결과 저장
- 추출 실행 이력 저장
- 재추출 실행
- 사용자 수정 텍스트 저장
- 원본 추출 텍스트와 사용자 수정본 분리 보존
- `input_hash` 기반 `is_outdated` 판정
- 페이지별 텍스트 저장
- 규칙 기반 섹션 후보 저장
- 사용자 소유권 검사
- 프론트 이력서 상세 화면에서 추출/수정/재추출/이력 조회

## 상태

- `PENDING`: 추출 전 또는 준비 상태
- `PROCESSING`: 추출 실행 중
- `COMPLETED`: 텍스트 추출 성공
- `FAILED`: 일반 추출 실패
- `TEXT_NOT_FOUND`: 파일은 읽었지만 추출 가능한 텍스트 없음
- `OCR_REQUIRED`: PDF에 텍스트 레이어가 없어 OCR 필요

## 추출 정책

- PDF는 text literal 기반 best-effort 추출을 사용한다.
- 암호화된 PDF는 `RESUME_EXTRACTION_PDF_ENCRYPTED`로 실패 처리한다.
- 텍스트 레이어가 없는 PDF는 `OCR_REQUIRED` 상태로 기록한다.
- OCR은 v0.3.1에서 실행하지 않는다.
- DOCX는 `word/document.xml`의 텍스트 노드를 추출한다.
- DOCX는 실제 페이지 경계를 안정적으로 알 수 없으므로 단일 logical page로 저장한다.
- 섹션 후보는 AI 분석 결과가 아니라 규칙 기반 힌트다.

## API

- `POST /api/v1/resumes/{resumeId}/files/{fileId}/extraction`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/extraction`
- `PATCH /api/v1/resumes/{resumeId}/files/{fileId}/extraction`
- `POST /api/v1/resumes/{resumeId}/files/{fileId}/extraction/retry`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/extraction/runs`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/extraction/runs/{runId}`

## DB

- `resume_file_extractions`: 파일별 최신 추출 결과와 사용자 수정본
- `resume_extraction_runs`: 최초 추출 및 재추출 실행 이력

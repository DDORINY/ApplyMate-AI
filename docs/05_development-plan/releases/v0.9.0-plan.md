# ApplyMate AI v0.9.0 안정화·운영 준비 계획

## 1. 목적

`v0.9.0`은 v1.0.0 MVP 릴리스 전 마지막 안정화 단계다. 신규 대형 기능보다 이미 구현된 인증, 커리어 프로필, 채용공고, 분석, 지원 문서, 지원 현황, 일정, 외부 연동, 추천, 알림 기능의 신뢰성을 높이는 데 집중한다.

## 2. 기준 상태

- 최신 완료 버전: `v0.8.0`
- 기준 브랜치: `main`
- 작업 브랜치: `feature/v0.9.0-stability-hardening`
- 현재 migration head: `20260720_0200`
- 주요 완료 영역: in-app 알림, 알림 설정, 알림 delivery, 알림 처리 run, mock email provider, worker dry run

## 3. 작업 범위

### Backend

- 주요 API의 인증·소유권 검사를 재점검한다.
- 목록 API의 pagination, filtering, ordering 동작을 검증한다.
- worker 작업의 idempotency, retry, 실패 기록을 보강한다.
- rate limit 적용 후보를 정리하고 우선순위가 높은 인증/AI/외부 연동 API부터 적용한다.
- 오류 코드와 실제 예외 처리의 불일치를 정리한다.

### Frontend

- 핵심 사용자 흐름의 빈 상태, 로딩 상태, 오류 상태를 점검한다.
- 로그인 후 대시보드, 공고, 추천, 일정, 알림, 지원 문서 흐름을 E2E 대상으로 묶는다.
- 모바일 화면에서 주요 카드, 목록, 폼, 내비게이션 깨짐을 수정한다.
- 접근성 기본 항목(label, focus, keyboard navigation)을 점검한다.

### Database

- 자주 조회되는 사용자 소유 데이터의 index 필요 여부를 확인한다.
- 알림, 추천, 일정, 지원 현황 조회 성능을 점검한다.
- migration head가 단일 head인지 검증한다.

### External integrations

- Google Calendar 실제 연결 상태와 실패 처리 경로를 검증한다.
- Gmail 분석 후보 생성 흐름의 mock/disabled/live provider 분기를 문서와 맞춘다.
- SMTP 실제 발송은 별도 계정과 샌드박스 환경에서 `NEEDS_VERIFICATION`을 해소한다.

## 4. 테스트 계획

- Backend 전체 테스트
- Frontend lint
- Frontend type check
- Frontend build
- 핵심 E2E 테스트 추가
- Docker Compose config 검증
- Alembic upgrade/downgrade 검증
- worker dry run 검증

## 5. 보안 점검

- `.env`와 secret 값이 저장소에 포함되지 않았는지 재확인한다.
- JWT, OAuth token, 외부 API key가 로그에 노출되지 않는지 확인한다.
- 업로드 파일, 외부 URL, AI prompt 입력에서 위험 입력값을 재점검한다.
- 사용자 소유 데이터 조회에 `user_id` 조건이 누락된 경로가 없는지 확인한다.

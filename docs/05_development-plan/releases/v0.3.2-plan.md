# v0.3.2 AI 이력서 구조화 분석 상세 개발계획

## 1. 버전 목적

v0.3.1에서 추출한 이력서 텍스트를 기반으로 AI가 이력서 핵심 정보를 구조화한다.

## 2. 사용자 가치

사용자는 업로드한 이력서에서 기술, 경력, 프로젝트, 학력, 자격증, 성과 후보를 빠르게 확인하고 기존 커리어 프로필과 비교할 수 있다.

## 3. 선행 조건

- 이력서 파일 업로드 완료
- 텍스트 추출 상태 `COMPLETED`
- 분석 입력 텍스트가 비어 있지 않음
- AI Provider가 `mock` 또는 `openai`로 사용 가능

## 4. 구현 범위

- 분석 입력 선택: `edited_text` 우선, 없으면 `raw_text`
- 구조화 schema, prompt, provider method
- 분석 결과와 실행 이력 DB 저장
- 사용자 수정본 분리 저장
- 재분석
- profile candidates 제공
- 상세 화면 AI 분석 UI

## 5. 제외 범위

- OCR 실행
- 프로필 자동 반영
- 자기소개서/지원 문서 생성
- 실제 OpenAI 유료 호출 강제 검증

## 6. 사용자 흐름

이력서 상세 화면 진입 → 추출 완료 파일 선택 → AI 분석 실행 → 구조화 결과 확인 → evidence 확인 → 수정본 저장 → 프로필 반영 후보 확인 → 필요 시 재분석

## 7. 기능 요구사항

- 분석 결과는 원문 근거에 기반해야 한다.
- 원문에 없는 경력, 기술, 성과, 기간은 생성하지 않는다.
- 중복 실행은 차단한다.
- 분석 실행 이력은 보존한다.
- 분석 입력이 변경되면 outdated로 표시한다.

## 8. 비기능 요구사항

- 사용자 소유권 검증
- Provider secret 로그 금지
- 프론트 로딩/빈/오류/처리 중 상태 표시
- 모바일 대응

## 9. DB 설계

- `resume_analyses`: 파일별 최신 분석 결과
- `resume_analysis_runs`: 모든 분석 시도 이력
- migration: `20260719_1700_create_resume_analysis_tables.py`

## 10. API 설계

- `POST /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `PATCH /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `DELETE /api/v1/resumes/{resumeId}/files/{fileId}/analysis`
- `POST /api/v1/resumes/{resumeId}/files/{fileId}/analysis/retry`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/runs`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/runs/{runId}`
- `GET /api/v1/resumes/{resumeId}/files/{fileId}/analysis/profile-candidates`
- `GET /api/v1/ai/resume-providers`

## 11. Backend 설계

- Model: `ResumeAnalysis`, `ResumeAnalysisRun`
- Schema: `ResumeAnalysisStructuredData`, public/update/run schemas
- Repository: `ResumeAnalysisRepository`
- Service: `ResumeAnalysisService`
- Provider: existing common AI Provider 확장
- Router: resumes endpoint에 파일 기준 analysis route 추가

## 12. AI Provider 설계

- `analyze_resume()` interface 추가
- Disabled Provider: 명확한 비활성 오류
- Mock Provider: deterministic result와 evidence 제공
- OpenAI Provider: Responses API 기반 structured JSON 검증

## 13. Prompt 및 출력 Schema

- 이력서 텍스트를 명령이 아닌 untrusted data로 취급한다.
- 출력은 Pydantic schema로 검증한다.
- 핵심 항목은 evidence를 포함해야 한다.

## 14. Frontend 설계

- `/resumes/{resumeId}` 상세 카드에 AI 분석 섹션 추가
- 분석 실행, 재분석, 삭제, JSON 수정 저장
- Provider/model/status/outdated/user edited 표시
- skills, summary, profile candidates, run history 표시

## 15. 보안 정책

- 모든 분석 API는 Access Token 필요
- resume_id/file_id/user_id 소유권 검증
- Provider API key와 원문 전체 응답 로그 금지
- Prompt injection 방어 문구 사용

## 16. 오류 처리

- `RESUME_ANALYSIS_EXTRACTION_REQUIRED`
- `RESUME_ANALYSIS_EXTRACTION_NOT_READY`
- `RESUME_ANALYSIS_TEXT_EMPTY`
- `RESUME_ANALYSIS_ALREADY_PROCESSING`
- `RESUME_ANALYSIS_INVALID_OUTPUT`
- `RESUME_ANALYSIS_EVIDENCE_INVALID`
- `RESUME_ANALYSIS_PROVIDER_UNAVAILABLE`

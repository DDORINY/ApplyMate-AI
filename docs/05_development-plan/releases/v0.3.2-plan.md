# v0.3.2 이력서 구조화 분석 계획

## 목표

v0.3.1에서 추출한 이력서 텍스트를 기반으로 AI가 이력서의 핵심 정보를 구조화한다.

## 범위

- 이력서 추출 결과의 `raw_text` 또는 `edited_text`를 분석 입력으로 사용한다.
- 기술, 경력, 프로젝트, 학력, 키워드, 요약, headline 후보를 구조화한다.
- 분석 결과에는 가능한 경우 근거 문장을 함께 저장한다.
- 사용자가 편집한 추출 텍스트가 있으면 원문보다 편집본을 우선한다.
- 입력 해시를 저장하여 추출 텍스트가 변경된 경우 outdated 상태를 표시한다.
- Mock Provider와 OpenAI Provider 모두 동일한 구조화 스키마를 사용한다.

## 제외 범위

- 이력서 내용을 사용자 커리어 프로필에 자동 반영하지 않는다.
- 사용자의 실제 경험에 없는 수치, 성과, 회사명, 기간을 생성하지 않는다.
- 이미지 기반 PDF OCR은 v0.3.2 범위가 아니다.
- 자기소개서 또는 맞춤형 지원 문서 생성은 v0.3.3 이후로 분리한다.

## API 초안

| Method | Path | 설명 |
| --- | --- | --- |
| POST | `/api/v1/resumes/{resume_id}/files/{file_id}/analysis` | 이력서 구조화 분석 실행 |
| GET | `/api/v1/resumes/{resume_id}/files/{file_id}/analysis` | 최신 분석 결과 조회 |
| PATCH | `/api/v1/resumes/{resume_id}/files/{file_id}/analysis` | 분석 결과 사용자 수정 |
| POST | `/api/v1/resumes/{resume_id}/files/{file_id}/analysis/retry` | 분석 재실행 |
| GET | `/api/v1/resumes/{resume_id}/files/{file_id}/analysis/runs` | 분석 실행 이력 조회 |

## 초기 구현 상태

- `ResumeAnalysisStructuredData` Pydantic 스키마 추가
- 이력서 분석 전용 프롬프트 템플릿 추가
- AI Provider에 `analyze_resume` 인터페이스 추가
- Mock Provider의 이력서 분석 응답 구조 추가
- OpenAI Provider의 이력서 분석 호출 진입점 추가
- Provider 단위 테스트 추가

## 완료 기준

- DB migration으로 최신 분석 결과와 실행 이력 저장
- 사용자 소유권 검증
- 추출 결과가 없거나 OCR_REQUIRED인 경우 명확한 오류 반환
- API 명세, DB 문서, 릴리스 문서 갱신
- Backend test, Frontend lint/type-check/build, Docker Compose 검증 통과

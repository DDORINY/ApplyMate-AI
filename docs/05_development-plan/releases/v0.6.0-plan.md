# ApplyMate AI v0.6.0 개발 계획 - 일일 맞춤 채용공고 추천

## 1. 버전 목적

v0.6.0은 사용자가 등록한 채용공고, 커리어 프로필, 이력서 분석 결과, 공고 적합도 분석, 지원 이력, 선호 조건을 바탕으로 일일 맞춤 채용공고 추천 기반을 구현한다.

이 버전의 추천은 외부 사이트 자동 크롤링이 아니라, ApplyMate AI 내부에 저장된 사용자 소유 데이터와 사용자가 등록한 공고를 기준으로 한다.

## 2. 핵심 원칙

- 사용자의 실제 경력과 저장된 분석 결과를 근거로 추천한다.
- 추천 사유와 부족한 점을 함께 제공한다.
- 지원 완료, 거절, 숨김 처리한 공고는 반복 추천하지 않는다.
- 추천 결과는 확정적 합격 가능성이 아니라 의사결정 보조 정보로 표현한다.
- 외부 채용 사이트 대규모 크롤링은 구현 범위에서 제외한다.

## 3. 포함 범위

- 추천 설정 관리
- 추천 후보 생성
- 추천 점수 계산
- 일일 추천 목록 조회
- 추천 사유 및 근거 저장
- 추천 제외/숨김 처리
- 추천 피드백 저장
- 대시보드 추천 요약 표시
- 관련 API, DB, 테스트, 문서 최신화

## 4. 제외 범위

- 외부 채용공고 자동 수집 크롤러
- 외부 채용 플랫폼 지원 자동화
- 유료 AI API 기반 대량 추천
- 합격 가능성 확정 예측
- 사용자 승인 없는 지원 상태 자동 변경

## 5. 추천 입력 데이터

- 커리어 프로필
- 경력, 프로젝트, 기술 스택
- 이력서 텍스트 추출 결과
- 이력서 구조화 분석 결과
- 등록된 채용공고
- 채용공고 AI 분석 결과
- 사용자-공고 적합도 분석 결과
- 지원 현황과 지원 결과
- 사용자 선호 조건
- 추천 피드백

## 6. 추천 결과 데이터

추천 결과는 다음 정보를 포함한다.

- 추천 대상 공고
- 추천 점수
- 추천 등급
- 주요 추천 사유
- 주요 리스크
- 근거 데이터
- 추천 생성일
- 사용자 피드백 상태

## 7. 추천 점수 정책

초기 버전은 규칙 기반 점수와 기존 적합도 분석 결과를 조합한다.

- 기술 일치도
- 경력 연차 적합성
- 프로젝트 경험 관련성
- 직무 키워드 적합성
- 희망 조건 일치도
- 마감 임박도
- 지원 이력 중복 여부
- 사용자의 숨김/관심 피드백

AI 기반 추천 문장 생성은 선택적으로 사용할 수 있으나, 추천 후보 산정의 필수 조건으로 두지 않는다.

## 8. 추천 상태

```text
RECOMMENDED
VIEWED
SAVED
HIDDEN
APPLIED
EXPIRED
```

## 9. API 계획

Base URL은 `/api/v1`을 유지한다.

- `GET /job-recommendations`
- `POST /job-recommendations/generate`
- `GET /job-recommendations/{recommendationId}`
- `POST /job-recommendations/{recommendationId}/save`
- `POST /job-recommendations/{recommendationId}/hide`
- `POST /job-recommendations/{recommendationId}/feedback`
- `GET /job-recommendation-settings`
- `PATCH /job-recommendation-settings`

## 10. DB 계획

예상 테이블은 다음과 같다.

- `job_recommendation_settings`
- `job_recommendation_runs`
- `job_recommendations`
- `job_recommendation_feedback`

모든 추천 데이터는 `user_id`를 기준으로 소유권 검사를 적용한다.

## 14. 릴리스 주의사항

- 추천 품질은 저장된 사용자 데이터와 공고 분석 데이터 품질에 의존한다.
- 외부 공고 수집은 v0.6.0 범위가 아니므로, 추천 대상은 기존 등록 공고로 제한한다.
- AI_PROVIDER가 `mock` 또는 `disabled`인 경우 추천 문장 생성은 규칙 기반 문구로 대체한다.

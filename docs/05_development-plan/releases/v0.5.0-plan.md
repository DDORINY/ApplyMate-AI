# v0.5.0 Google Calendar Integration Plan

## 1. 버전 목적

ApplyMate AI 내부 일정과 Google Calendar를 연결할 수 있는 기반을 구현한다. 로그인 OAuth와 Calendar OAuth를 분리하고, 사용자가 명시적으로 연결한 Google 계정에 대해서만 Calendar 권한을 사용한다.

## 2. 사용자 가치

사용자는 ApplyMate AI에서 관리하는 지원 마감, 면접, 코딩테스트, 과제, 결과 발표 일정을 개인 Google Calendar에서도 확인할 수 있다.

## 3. 선행 조건

- v0.4.1 일정 관리 완료
- v0.4.2 대시보드 완료

- 로컬/테스트 환경은 Google 실 credentials 없이 mock provider로 검증한다.

## 6. OAuth 정책

- 로그인용 OAuth와 Calendar 연동용 OAuth를 같은 기능으로 취급하지 않는다.
- Calendar scope는 사용자가 `/settings/accounts`에서 명시적으로 연결할 때만 요청한다.
- Calendar 연결 상태, 권한, 마지막 동기화 상태를 사용자에게 표시한다.
- OAuth token과 refresh token은 로그에 남기지 않는다.

## 7. 권한/Scope

초기 후보:

- `https://www.googleapis.com/auth/calendar.events`

실제 scope는 구현 시 Google 공식 문서 기준으로 재확인한다.

## 13. 보안

- Google client secret은 저장소에 커밋하지 않는다.
- token은 암호화해서 저장한다.
- refresh token은 일반 로그와 API 응답에 노출하지 않는다.
- 연결 해제 시 내부 일정은 보존하고 외부 연결 정보만 비활성화한다.

## 14. 동기화 정책

- 내부 일정 저장 성공과 외부 Calendar 동기화 성공을 분리한다.
- 외부 동기화 실패가 내부 일정 저장 실패로 이어지지 않도록 한다.
- 외부 event id를 저장해 이후 수정/취소에 사용한다.
- 사용자가 연결을 해제하면 자동 동기화를 중단한다.

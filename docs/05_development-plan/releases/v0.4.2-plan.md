# v0.4.2 Dashboard Plan

## Purpose

v0.4.2 will add the ApplyMate AI dashboard. The dashboard should summarize the user's current job search state across applications, schedules, AI analyses, matches, and generated documents.

## User value

The user can open one page and immediately understand what needs attention today, this week, and next in their job search flow.

## Prerequisites

- v0.4.0 application tracking
- v0.4.1 schedule management
- Existing job, resume, AI analysis, matching, and document data

## Scope

- Dashboard summary page
- Application status statistics
- Today and this-week schedules
- Due-soon jobs and application deadlines
- Preparing applications
- Recent AI job analyses
- Recent job match results
- Recent application documents
- Recent activity feed

## Exclusions

- New AI generation features
- External calendar sync
- Gmail ingestion
- Push/email delivery
- Complex BI/reporting charts

## Candidate API

Base path: `/api/v1/dashboard`

- `GET /summary`
- `GET /applications/status-counts`
- `GET /schedules/today`
- `GET /schedules/week`
- `GET /jobs/due-soon`
- `GET /activity`

## Candidate frontend route

- `/dashboard`

The home page may link to the dashboard or become the dashboard after the navigation policy is finalized.

## Completion criteria

- Backend API and tests
- Frontend dashboard screen
- Loading, empty, and error states
- Documentation updates
- Frontend lint/type-check/build
- Backend ruff/pytest
- Docker compose config

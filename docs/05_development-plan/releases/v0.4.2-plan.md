# v0.4.2 Dashboard Development Plan

## 1. 버전 목적

ApplyMate AI의 지원 준비 현황을 한 화면에서 요약하는 읽기 전용 대시보드를 제공한다.

## 2. 사용자 가치

사용자는 오늘 처리해야 할 일정, 이번 주 일정, 마감 임박 항목, 준비 중인 지원, 최근 AI 분석과 문서를 빠르게 확인할 수 있다.

## 3. 선행 조건

- v0.4.0 지원 현황 관리 완료
- v0.4.1 일정 관리 완료
- 현재 migration head: `20260719_2000`

## 4. 구현 범위

- 대시보드 집계 API
- `/dashboard` 프론트 화면
- 지원 상태 그룹 집계
- 오늘/이번 주 일정
- 다가오는 일정 마감
- 마감 임박 공고
- 준비 중인 지원 항목
- 최근 AI 분석/적합도/이력서 분석/지원 문서
- 최근 활동

## 5. 제외 범위

- 대시보드에서의 데이터 생성/수정/삭제
- 위젯 개인화
- 고급 차트 라이브러리
- Google Calendar 동기화
- Gmail 연동
- 별도 activity log 테이블

## 6. 사용자 흐름

1. 사용자가 로그인한다.
2. 헤더에서 `/dashboard`로 이동한다.
3. 요약 카드와 상태 분포를 확인한다.
4. 오늘/이번 주 일정과 마감 항목을 확인한다.
5. 최근 분석/문서/활동 항목에서 상세 화면으로 이동한다.

## 7. 대시보드 위젯 정의

- Summary cards
- Application status distribution
- Application activity
- Today events
- Week events
- Upcoming deadlines
- Due-soon jobs
- Preparing applications
- Recent job analyses
- Recent matches
- Recent resume analyses
- Recent documents
- Recent activities

## 8. 기간 필터

- `period=7d|30d|90d|all`
- `start_date`, `end_date`가 함께 제공되면 사용자 지정 기간으로 집계한다.
- `timezone` 기본값은 `Asia/Seoul`이다.
- 기본 기간은 `30d`이다.
- 오늘/이번 주/마감 위젯은 각 위젯 고유의 기간을 사용한다.

## 9. 집계 기준

| Dashboard group | Application statuses |
| --- | --- |
| `PREPARING` | `SAVED`, `PREPARING` |
| `APPLIED` | `APPLIED` |
| `IN_PROGRESS` | `DOCUMENT_REVIEW`, `CODING_TEST`, `ASSIGNMENT` |
| `INTERVIEW` | `INTERVIEW`, `FINAL_INTERVIEW` |
| `OFFER` | `OFFER` |
| `REJECTED` | `REJECTED` |
| `WITHDRAWN` | `WITHDRAWN` |
| `CLOSED` | `CLOSED` |

상태 그룹 계산은 백엔드 책임이다.

## 10. Backend 설계

- `backend/app/schemas/dashboard.py`
- `backend/app/repositories/dashboard.py`
- `backend/app/services/dashboard.py`
- `backend/app/api/v1/endpoints/dashboard.py`
- `backend/tests/test_dashboard.py`

Router는 인증 사용자와 query parameter만 처리하고, 집계 로직은 service/repository로 분리한다.

## 11. API 설계

- `GET /api/v1/dashboard`

Query:

- `period`
- `start_date`
- `end_date`
- `timezone`
- `recent_limit`

Response:

- `summary`
- `application_status_counts`
- `application_activity`
- `today_events`
- `week_events`
- `upcoming_deadlines`
- `due_soon_jobs`
- `preparing_applications`
- `recent_job_analyses`
- `recent_matches`
- `recent_resume_analyses`
- `recent_documents`
- `recent_activities`
- `generated_at`
- `timezone`

## 12. Query optimization

- 위젯별 제한된 query를 사용한다.
- 전체 row를 메모리에 적재하지 않는다.
- 관계 데이터는 필요한 범위에서 `selectinload`를 사용한다.
- 기존 인덱스를 우선 사용하고, v0.4.2에서는 신규 migration을 만들지 않는다.

## 13. Frontend 설계

- `/dashboard` 페이지 추가
- TanStack Query로 서버 상태 관리
- 새 차트 라이브러리 없이 CSS bar로 상태 분포 표시
- 카드/목록 기반 responsive layout
- 각 항목은 가능한 경우 원본 상세 화면으로 링크한다.

## 14. Empty/Error states

- 데이터가 없으면 빈 상태 문구를 표시한다.
- 인증 토큰이 없으면 API client에서 로그인 필요 오류를 발생시킨다.
- API 오류는 대시보드 상단 패널에 표시한다.
- 로딩 중에는 skeleton panel을 표시한다.

## 15. Ownership

- 모든 query는 `current_user.id`를 기준으로 제한한다.
- 다른 사용자의 공고, 지원, 일정, 분석, 문서는 대시보드에 노출하지 않는다.

## 16. Error handling

- `DASHBOARD_INVALID_TIMEZONE`
- `DASHBOARD_INVALID_DATE_RANGE`
- 일반 validation 오류는 `VALIDATION_ERROR`

## 17. Test strategy

## 21. 알려진 위험

- 대시보드 위젯이 많아 query 수가 늘어날 수 있다.
- 실제 운영 데이터 규모에서는 추가 인덱스가 필요할 수 있다.
- 외부 AI/OpenAI 실제 호출 검증은 운영 key/model이 있어야 가능하다.
- Google Calendar 실제 동기화는 v0.5.0 범위다.

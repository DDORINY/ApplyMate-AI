# v0.7.0 AI 자기소개서 개선 루프 상세 개발계획

## 1. 목적

v0.7.0은 기존 지원 문서 생성 기능을 확장해 사용자가 문장 단위로 개선 요청을 남기고, 변경 전후를 비교한 뒤 승인한 내용만 새 버전으로 저장하는 자기소개서 개선 루프를 구현한다.

## 2. 사용자 가치

사용자는 이미 생성한 자기소개서를 다시 처음부터 만들지 않고, 부족한 문장이나 문단만 근거를 유지한 채 개선할 수 있다.

## 3. 구현 범위

- 지원 문서 개선 요청 생성
- 문장/문단 단위 개선 대상 지정
- 사용자 피드백 저장
- 개선 후보 생성
- 변경 전후 비교
- 근거 유지 여부 표시
- 사용자 승인 후 새 문서 버전 저장
- 개선 실행 이력
- API/DB/Frontend/문서/테스트

## 4. 제외 범위

- 사용자 경험에 없는 내용 생성
- 합격 가능성 확정 표현
- 기업 정보 임의 추정
- 사용자 승인 없는 문서 버전 덮어쓰기
- 실제 외부 제출

## 5. 데이터 기준

개선 후보는 다음 데이터를 근거로 한다.

- 기존 지원 문서 버전
- 기존 문서 출처
- 사용자 커리어 프로필
- 경력, 프로젝트, 기술
- 등록된 채용공고
- 채용공고 분석
- 사용자 피드백

## 6. Backend 초안

- 개선 요청 table
- 개선 후보 table
- 개선 실행 run table
- 근거 연결 table
- 기존 generation service 재사용
- AI provider는 기존 abstraction 유지
- mock provider 테스트 우선

## 7. API 초안

- `POST /api/v1/documents/{documentId}/improvement-runs`
- `GET /api/v1/documents/{documentId}/improvement-runs`
- `GET /api/v1/documents/improvement-runs/{runId}`
- `POST /api/v1/documents/improvement-runs/{runId}/approve`
- `POST /api/v1/documents/improvement-runs/{runId}/reject`

## 8. Frontend 초안

- 문서 상세에서 개선 요청 CTA
- 개선 요청 폼
- 문장/문단 선택
- 변경 전후 비교 카드
- 근거 유지 표시
- 승인 후 새 버전 생성

## 9. 테스트 계획

- 문서 소유권 검증
- 개선 요청 생성
- mock provider 후보 생성
- 근거 없는 내용 차단
- 승인 시 새 버전 생성
- 거절 시 기존 버전 유지
- frontend lint/type-check/build

## 10. 완료 기준

- 개선 루프 기능 구현
- 테스트 통과
- 문서/API/DB 최신화
- 사용자의 실제 경험에 없는 내용을 생성하지 않음
- 승인 전 기존 문서 버전을 변경하지 않음

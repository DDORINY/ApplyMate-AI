# v0.3.3 맞춤형 지원 문서 생성 계획

## 1. 버전 목적

사용자 커리어 프로필, 이력서 분석 결과, 채용공고 분석 결과, 사용자-공고 적합도 결과를 근거로 맞춤형 지원 문서 초안을 생성한다.

## 2. 입력 데이터

- 사용자 커리어 프로필
- 이력서 추출/분석 결과
- 채용공고 분석 결과
- 사용자-공고 적합도 분석 결과
- 사용자가 직접 확인한 추가 사실

## 3. 생성 대상

- 자기소개서 문항별 초안
- 지원 동기 초안
- 직무 적합성 요약
- 강점/보완점 기반 문장 후보
- 근거 기반 bullet point

## 4. 핵심 원칙

- 사용자 실제 경험에 없는 내용을 생성하지 않는다.
- 수치, 성과, 기간, 회사명, 프로젝트명은 근거가 있을 때만 사용한다.
- 생성 문장마다 가능한 경우 근거를 연결한다.
- 확정적 합격 가능성 표현을 금지한다.
- 사용자가 최종 검토해야 함을 명시한다.

## 5. DB 초안

- `application_documents`
  - 최신 문서 초안
  - job_posting_id, resume_analysis_id, job_match_id 연결
  - structured_result, edited_result, status, input_hash
- `application_document_runs`
  - 생성 실행 이력
  - provider, model, prompt_version, schema_version, usage metadata

## 6. API 초안

- `POST /api/v1/jobs/{jobId}/application-documents`
- `GET /api/v1/jobs/{jobId}/application-documents`
- `PATCH /api/v1/jobs/{jobId}/application-documents/{documentId}`
- `DELETE /api/v1/jobs/{jobId}/application-documents/{documentId}`
- `POST /api/v1/jobs/{jobId}/application-documents/{documentId}/retry`
- `GET /api/v1/jobs/{jobId}/application-documents/{documentId}/runs`

## 7. AI 설계

- 기존 AI Provider 구조를 재사용한다.
- Prompt는 별도 템플릿으로 관리한다.
- 출력은 Pydantic schema로 검증한다.
- Prompt injection 방어를 포함한다.

## 8. Frontend 설계

- 채용공고 상세 화면에 지원 문서 생성 섹션 추가
- 이력서 분석 선택
- 문항 입력/관리
- 초안 생성, 수정, 재생성
- 근거 보기

## 9. 테스트 전략

- Mock Provider 기반 deterministic 문서 생성
- 근거 없는 내용 차단
- 사용자 소유권 검증
- 입력 데이터 누락 차단
- 사용자 수정본 보존
- 실행 이력 보존

## 10. 제외 범위

- 외부 채용 사이트 자동 제출
- 실제 이메일 발송
- Google Docs/Word export
- 합격 가능성 예측

## 11. 완료 기준

- Backend test 통과
- Frontend lint/type-check/build 통과
- migration upgrade/downgrade 검증

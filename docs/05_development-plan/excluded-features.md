# 제외 기능

이 문서는 현재 프로젝트 범위에서 명시적으로 제외했거나 MVP 기본 기능으로 넣지 않는 항목을 정리한다.

| 기능 | 분류 | 제외 이유 | 현재 대체 방식 | 재검토 조건 | 예상 버전 |
| --- | --- | --- | --- | --- | --- |
| Kakao/Naver/Apple 로그인 | MVP 제외 | Google/GitHub 우선 | 이메일/Google/GitHub 로그인 | 사용자 요구 증가 | v1.x 이후 |
| SMS 인증 | 정책상 제외 | 비용/개인정보/운영 복잡도 | 이메일 인증 | 휴대폰 인증이 필수 요구가 될 때 | 미정 |
| WebAuthn | MVP 제외 | 구현/UX 복잡도 | 비밀번호+OAuth | 보안 요구 상승 | v1.x 이후 |
| TOTP | MVP 제외 | 초기 사용자 가치 대비 비용 높음 | 세션 관리/보안 이벤트 | 관리자/고보안 계정 필요 | v1.x 이후 |
| CAPTCHA 우회 | 영구 제외 | 보안 우회 목적 기능 | 없음 | 재검토 없음 | 없음 |
| 무단 대규모 crawling | 정책상 제외 | 약관/robots/법적 리스크 | 사용자 입력, URL 단건 등록 | 공식 API/RSS 확보 | 미정 |
| Provider access token 저장 | 정책상 제외 | 외부 계정 권한/보안 리스크 | 로그인 식별 정보만 저장 | 연동 기능별 최소 scope 설계 | v0.5.x |
| 운영 Secret 저장소 커밋 | 영구 제외 | 보안 위반 | `.env.example`만 관리 | 재검토 없음 | 없음 |
| AI 단독 숫자 점수 계산 | 정책상 제외 | 근거/재현성 부족 | 규칙 기반 deterministic 계산 | 검증 가능한 보조 모델 설계 | 미정 |
| Celery 조기 도입 | 기술적 제외 | 현재 동기 처리로 충분 | FastAPI 직접 처리 | 대량/장시간 작업 증가 | v0.5.x 이후 |
| pgvector 조기 도입 | 기술적 제외 | 현재 추천/검색 단계 전 | 관계형 DB/규칙 기반 | 이력서/공고 embedding 필요 | v0.3.x 이후 |

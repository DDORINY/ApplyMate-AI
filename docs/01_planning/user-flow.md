# 사용자 흐름

## v0.1.1 인증 흐름

### 회원가입

1. 사용자가 `/signup` 화면에 접근한다.
2. 이름, 이메일, 비밀번호, 비밀번호 확인을 입력한다.
3. Frontend가 이메일 형식, 비밀번호 길이, 비밀번호 확인 일치를 검증한다.
4. Backend가 이메일을 소문자와 공백 제거 기준으로 정규화한다.
5. Backend가 중복 이메일을 확인한다.
6. Backend가 비밀번호를 해시로 저장한다.
7. 회원가입 성공 후 사용자는 로그인 화면으로 이동한다.

### 로그인

1. 사용자가 `/login` 화면에 접근한다.
2. 이메일과 비밀번호를 입력한다.
3. Backend가 사용자 존재 여부와 비밀번호를 검증한다.
4. 비활성 사용자는 로그인할 수 없다.
5. 로그인 성공 시 Access Token을 응답 본문으로 반환한다.
6. Refresh Token은 HttpOnly Cookie로 설정된다.
7. Frontend는 Access Token을 저장하고 `/me` 화면으로 이동한다.

### 로그인 상태 확인

1. 사용자가 `/me` 보호 화면에 접근한다.
2. Frontend가 저장된 Access Token으로 `/api/v1/auth/me`를 호출한다.
3. Access Token이 유효하면 사용자 정보를 표시한다.
4. Access Token이 만료되면 `/api/v1/auth/refresh`로 재발급을 시도한다.
5. 재발급이 실패하면 로그인 화면으로 이동한다.

### 로그아웃

1. 사용자가 `/me` 화면에서 로그아웃을 선택한다.
2. Backend가 현재 Refresh Token을 폐기한다.
3. Backend가 Refresh Token Cookie를 제거한다.
4. Frontend가 Access Token을 삭제한다.
5. 사용자는 로그인 화면으로 이동한다.

## 향후 MVP 사용자 흐름

### 커리어 프로필 작성

1. 사용자가 로그인한다.
2. 희망 직무, 기술 스택, 경력 구분을 등록한다.
3. 프로젝트와 경력 상세를 입력한다.
4. 선호 조건과 제외 조건을 저장한다.
5. 등록된 프로필은 공고 분석과 문서 생성의 근거 데이터로 사용된다.

### 채용공고 등록 및 분석

1. 사용자가 채용공고 URL 또는 본문을 입력한다.
2. 시스템이 회사명, 직무, 주요 업무, 필수 조건, 우대 조건을 구조화한다.
3. 마감일과 전형 일정이 있으면 일정 후보로 저장한다.
4. 사용자는 분석 결과를 확인하고 수정한다.

### 지원 문서 생성

1. 사용자가 채용공고와 이력서 또는 커리어 프로필을 선택한다.
2. 시스템이 공고 요구사항과 사용자 경험을 비교한다.
3. AI가 사용자 경험을 근거로 지원 문서 초안을 생성한다.
4. 사용자는 생성 근거를 확인하고 문서를 수정한다.
5. 최종 문서는 공고별 버전으로 저장된다.

### 일정 관리

1. 사용자가 공고 마감일, 과제 제출일, 면접 일정을 등록한다.
2. 일정은 `CONFIRMED`, `ESTIMATED`, `USER_INPUT`으로 구분된다.
3. Google Calendar 연결 시 사용자의 승인 후 외부 캘린더에 등록한다.
4. 외부 동기화 실패가 내부 일정 저장 실패로 이어지지 않도록 처리한다.
